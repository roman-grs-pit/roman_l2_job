Full run — design

Footprint, parameters, and measured zodiacal background

Science motivation

This configuration was built to support simulated grism observations of a Wide-Field2 sub-region near (RA = 10°, Dec = 0°), at four position angles (PA = 0°, 10°, 170°, 180°). We need imaging coverage of the grism reference position and a few arcminutes along the dispersion direction so that source positions measured from imaging constrain the spectral trace extraction.

Footprint filter

The pointings file (pointings_full.ecsv) is a subset of catalogs/HLWAS.sim.ecsv (the full HLWAS-survey simulated pointings) produced by scripts/filter_pointings.py:

region.type      = cone
region.ra        = 10.0°
region.dec       = 0.0°
region.radius    = 0.5°
bandpass         = F158
only_pass        = None
only_segment     = None
only_visit       = None

A visit survives the filter if any of its dithers lands inside the cone — matching visits contribute all their dithers (no partial inclusion). The only_pass/segment/visit nulls here mean no further restriction after the region test; the smoke config sets these to a single-visit subset.

The selection yields 6 visits × 3 dithers = 18 exposures across passes 15 and 16, segments 490 and 521:

The measured sky rate reflects the zodiacal background that the romanisim sim computed for this field and date: on-ecliptic (latitude −4°) and at a large solar elongation (~128°), which is a structurally bright but well-observed part of the sky for a Dec ≈ 0 target. Within the Roman observing constraint (elongation roughly 54°–126°), higher elongation minimises zodi for an ecliptic target; the sim date is already near the upper end of that window, so shifting dates cannot meaningfully lower the background from this pointing.

## Science motivation

This configuration was built to support simulated **grism observations** of a
Wide-Field2 sub-region near **(RA = 10°, Dec = 0°)**, at four position angles
(PA = 0°, 10°, 170°, 180°). We need imaging coverage of the grism reference
position and a few arcminutes along the dispersion direction so that source
positions measured from imaging constrain the spectral trace extraction.

## Footprint filter

The pointings file (`pointings_full.ecsv`) is a subset of
`catalogs/HLWAS.sim.ecsv` (the full HLWAS-survey simulated pointings)
produced by `scripts/filter_pointings.py`:

```{python}
#| echo: false
print(f"region.type      = {cone['type']}")

print(f"only_segment     = {cfg['pointings']['only_segment']}")

```

A visit survives the filter if *any* of its dithers lands inside the cone —
matching visits contribute *all* their dithers (no partial inclusion). The
`only_pass/segment/visit` nulls here mean no further restriction after the
region test; the `smoke` config sets these to a single-visit subset.

The selection yields **6 visits × 3 dithers = 18 exposures** across passes 15
and 16, segments 490 and 521:


## Pointings map (wide view)

```{python}
#| echo: false
#| fig-cap: "HLWAS F158 pointings near our field. Grey dots are the full HLWAS F158 catalog in a 5° window around our target; red `+` are the 18 selected for the full run; the 0.5° filter cone is drawn as a dashed circle; the grism target (10°, 0°) is the red star."
hlwas = Table.read(REPO / "catalogs/HLWAS.sim.ecsv", format="ascii.ecsv")
hlwas_f158 = hlwas[hlwas["BANDPASS"] == cfg["pointings"]["bandpass"]]
c0 = SkyCoord(ra=cone["ra"]*u.deg, dec=cone["dec"]*u.deg)
c_all = SkyCoord(ra=np.asarray(hlwas_f158["RA"])*u.deg,
                 dec=np.asarray(hlwas_f158["DEC"])*u.deg)

def _draw_pointings(ax, radius_deg):
    near_mask = c0.separation(c_all).deg < radius_deg
    ax.scatter(hlwas_f158["RA"][near_mask], hlwas_f158["DEC"][near_mask],
               s=3, color="lightgray",
               label=f"{near_mask.sum()} HLWAS F158 pointings")
    ax.scatter(df_p["RA"], df_p["DEC"], s=80, color="tab:red", marker="+",
               lw=2, label=f"{len(df_p)} selected pointings")
    theta = np.linspace(0, 2*np.pi, 200)
    ax.plot(cone["ra"] + cone["radius_deg"]*np.cos(theta)
            / np.cos(np.deg2rad(cone["dec"])),
            cone["dec"] + cone["radius_deg"]*np.sin(theta),
            ls="--", color="k", lw=1, label=f"filter cone r={cone['radius_deg']}°")
    ax.plot(cone["ra"], cone["dec"], marker="*", color="tab:red",
            markersize=20, label="grism target")
    ax.set_xlabel("RA (deg)")
    ax.set_ylabel("Dec (deg)")
    ax.invert_xaxis()
    ax.set_aspect("equal")
    ax.grid(True, alpha=0.3)

fig, ax = plt.subplots(figsize=(8, 8))
_draw_pointings(ax, radius_deg=5.0)
ax.legend(loc="lower right")
ax.set_title("Pointings — 5° around target")
plt.show()
```

## Measured zodiacal background

Per-bandpass expected zodi rates for Roman WFI are documented on the
[Roman WFI technical page](https://science.nasa.gov/mission/roman-space-telescope/wfi-technical/);
our measurement below compares against that reference.

```{python}
#| echo: false
#| tbl-cap: "Sigma-clipped sky rate per SCA from one exposure. Gain is from the CRDS gain reference file (median ≈ 1.87 e⁻/DN)."
import roman_datamodels.datamodels as rdm
from astropy.stats import sigma_clipped_stats

GAIN_MEDIAN_EPERDN = 1.87
L2_GLOB = sorted((REPO / "output/cal").glob(
    "r0000101015521001006_0001_wfi*_f158_cal.asdf"))
rows = []
for p in L2_GLOB:
    m = rdm.open(p)
    data = np.asarray(m.data, dtype=np.float64)
    _, med, std = sigma_clipped_stats(data, sigma=3, maxiters=3)
    del m
    sca = int(str(p.name).split("wfi")[1][:2])
    rows.append({"SCA": sca, "sky_DN_per_s": med,
                 "sky_e_per_pix_per_s": med * GAIN_MEDIAN_EPERDN})
sky = pd.DataFrame(rows).sort_values("SCA")
sky.style.format({"sky_DN_per_s": "{:.4f}",
                  "sky_e_per_pix_per_s": "{:.3f}"})
```

well-observed part of the sky for a Dec ≈ 0 target. Within the Roman
observing constraint (elongation roughly 54°–126°), higher elongation
minimises zodi for an ecliptic target; the sim date is already near the
upper end of that window, so shifting dates cannot meaningfully lower the
