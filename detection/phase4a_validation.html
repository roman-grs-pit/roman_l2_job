Phase 4a — L2 + L3 validation on pair 11119

End-to-end pipeline run on one (pair, skycell) before scaling

What Phase 4a does

Before scaling to all 5 pairs, validate the full L2 → L3 → (Phase 5) source detection chain on one (pair, skycell). Concretely, for pair 11119 (skycell 000m18x60y42):

  1. Build a combined truth catalog (stars + galaxies merged in one parquet — romanisim ingests both via the per-row type column).
  2. Emit 22 romanisim-make-image commands from the corrected + HLWAS-expanded Phase 2 list (see below), each with a deterministic CRC32-of-basename seed so every dither draws an independent noise realisation.
  3. Run the sims 4-way parallel (~34 min). Output L2s in output/detection/l2/ (~200 MB each).
  4. Run skycell_asn --product-type full on the 22 L2s — target skycell gets all 22 members.
  5. Run MosaicPipeline → 503 MB coadd for x60y42.
  6. Inspect depth, weight, image, truth-to-image correspondence.

Wall time for pair 11119: ~40 min end-to-end (34 min sims + 3 min mosaic + <1 min validate).

Phase 2 WCS fix (applied before this run)

The first Phase-4a run surfaced a Phase-2 bug. Phase 2 had been computing SCA corners via galsim.roman.getWCS, which uses GalSim’s analytical (SIAF-based) distortion. The L2 sims run with --usecrds and use the CRDS distortion reference instead; the two models disagree by ~arcmin per SCA, flipping overlap decisions near skycell edges. Consequence: the first-run selected “depth 6” skycell x58y45 only received contributions from 4 of the 6 pointings at sim time.

Fix: use romanisim.wcs.get_wcs(meta_dict, usecrds=True) with fill_in_parameters(coord, pa_aper, boresight=False) — the exact code path romanisim-make-image --usecrds takes. After the fix, Phase-2 predictions and skycell_asn agree on every skycell.

Per-pixel depth — the pair alone isn’t enough; expand to HLWAS

The first corrected run simulated only the pair’s 6 pointings. That gave n_pointings = 6 on x60y42 but per-pixel depth of just 4: HLWAS dithers shift ~7-12′ between exposures — larger than the 4.6′ skycell — so the three dithers put three different SCAs over different regions of the skycell, not the same pixels. Most interior pixels get covered by only 2 of the 3 dithers (× 2 passes = depth 4).

Fix: at Phase 2’s end, sweep all of HLWAS F158 for any pointing within 0.6° of the selected skycell and keep (pointing, SCA) pairs that overlap (via the CRDS-backed WCS). For pair 11119’s skycell the expansion adds 16 more L2 files from nearby visits 9, 10, and 12 (and their pass-16 revisits) on top of the original 6 dithers from visit 11. All expanded sims use the pair’s SIM_DATE so zodi stays fixed across the mosaic — we get HLWAS-like depth at a single date.

With 22 L2 files the mosaic depth map reaches max 12, median interior 10:

depth tier fraction of skycell
depth 10-12 (core) ~55%
depth 8 ~30%
depth 4-6 (edge bands) ~12%
depth 0-2 (corners) ~3%

This matches what the full-run measured for its design-depth skycells (median ~6, max ~8 there because the full-run budget included the pair only, not neighbours). Our detection study gets deeper because we pull in every HLWAS visit whose SCAs overlap the target skycell.

Sigma-clipped median 0.356 MJy/sr is consistent with Phase-1’s galsim.getSkyLevel prediction of 0.96 e⁻/pix/s for this pair at its sim date (a unit conversion from e⁻/pix/s → MJy/sr puts this within 5% of the measured value — the same sanity check that held for the full run’s 0.49 MJy/sr measurement).

What works

  • L2 sims: 6/6 succeed in 11 minutes (4-way parallel, ~5.8 min/sim).
  • skycell_asn: target skycell gets all 6 L2 members after the Phase-2 WCS fix.
  • MosaicPipeline: clean 502 MB coadd, no rejection artifacts, uniform depth over the unique core.
  • Truth mapping: brightest star (mag 23.0) at its injected RA/Dec in the mosaic. The Sobol-spread sources are all inside the mosaic footprint.
  • Zodi: measured sigma-clipped median matches the Phase-1 galsim.getSkyLevel prediction.

Checkpoint

Phase 4a criteria met. Ready for Phase 4b scale-up (5 pairs × N skycells per pair). Per the user’s directive, if 4a passes we go to 5 skycells per pair for 4b — ~30 L2 files per pair × 5 = 150 L2 files, ~6 hr at 4-way parallelism.
